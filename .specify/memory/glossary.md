# WIMS-BFP Project Glossary
**Version:** 1.0
**Context:** Definitions for the Web-based Incident Management System for the Bureau of Fire Protection.

## 1. Domain Terminology (BFP & Public Safety)
- **BFP (Bureau of Fire Protection):** The government agency in the Philippines responsible for fire prevention, suppression, and emergency medical services.
- **FSOD (Fire Safety Operations Division):** The division responsible for fire incident investigation and safety enforcement.
- **AFOR (After Fire Operation Report):** The official legal document generated after a fire incident is suppressed.
- **Barangay:** The smallest administrative division in the Philippines; the primary granular unit for fire incident pinpointing.
- **Triage:** The process of sorting incoming community reports to determine their validity before dispatching official BFP resources.
- **Alarm Level:** The scale of fire severity (e.g., First Alarm, Second Alarm, Task Force Alpha).

## 2. Technical Architecture Concepts
- **Community Tier (Unverified):** The data layer representing incoming crowdsourced reports from citizens that have not yet been vetted by a BFP Validator.
- **Official Tier (Verified):** The "Single Source of Truth" database containing only incidents that have been officially confirmed and investigated.
- **Sovereign Trust:** The architectural principle that the system must remain operational and secure even if external third-party providers (like Google or Supabase) are unavailable.
- **SLM (Small Language Model):** Specifically refers to the **Qwen2.5-3B** model used for local forensic analysis of security logs.
- **XAI (Explainable AI):** The narrative output generated by the SLM to explain *why* a specific network packet was flagged as a threat (e.g., False Data Injection).
- **Soft Delete:** The practice of marking a record as `is_archived = TRUE` rather than physically removing it from the database to preserve legal evidence.

## 3. Security & Forensic Definitions
- **False Data Injection (FDI):** A cyber-attack where a malicious actor sends fake fire reports to the system to cause resource exhaustion or panic.
- **SID (Suricata Signature ID):** A unique numerical identifier for a specific security rule triggered in the Suricata Intrusion Detection System.
- **PKCE (Proof Key for Code Exchange):** The security extension to OAuth 2.0 used in our Keycloak implementation to prevent authorization code interception.
- **Chain of Custody:** The chronological documentation showing the seizure, custody, control, and analysis of data (tracked via `incident_verification_history`).
